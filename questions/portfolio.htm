Portfolio Risk Management

1. a. Given a list of stocks (maximum one share of any stock), create the most optimised investment portfolios for your clients

1. b. Given a list of stocks (with at least one share of each stock), create the most optimised investment portfolios for your clients. If the starting capital is not enough to buy at least one of each stock, there cannot be any profits.

1. c. Given a list of stocks (any number of share of any stocks), create the most optimised investment portfolios for your clients

2. Given a list of stocks (maximum one share of any stock), create the most optimised investment portfolios for your clients with risk contraints. Total risk must not exceed the client risk appetite

Requirements

  • Expose the following POST endpoints :

    • /maximize_1a
    • /maximize_1b
    • /maximize_1c
    • /maximize_2

  • Return the results as JSON

Constraints

  • 30 second timeout

Input and Output Description

The HTTP POST request will come with a body of Content-Type: application/json

1. a

Sample Input JSON

Stock has a ticker of type string, expectedProfit of type int, sharePrice of type int

{
        "startingCapital": 400,
        "stocks": [
            [
                "Sony", //ticker of type string
                30, //expected profit of type int
                400  //share price of type int
            ],
            [
                "Dell",
                20,
                300
            ],
            [
                "Disney",
                15,
                100
            ],
            [
                "Apple",
                20,
                100
            ]
        ]
    }

Sample output JSON

{
        "profit": 40,
        "portfolio": [
            "Apple", "Dell"  //ticker of the stocks, in any order
        ]
}
